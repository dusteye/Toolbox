1.学习Bash

[编辑]

主要内容

Bash基本命令，Shell脚本,expect 

掌握常用的场景应用、以及一些技巧：
cp、mv、ssh、scp、vi……

1.熟悉Linux下的各种常规命令
2.学会使用man文档
3.学会写简单的shell脚本
4.要求每个人把自己的学习心得都贴到本论坛的《Bash Tips》贴子下，要求每个人都要发贴。
5.在论坛提问之前请先阅读《提问的智慧》
[编辑]

时间

三天:7.16-7.18

OReilly - Running Linux 4th Edition
O'Reilly - Learning Debian GNU Linux
LinuxTips(Linux 一句话精彩问答)
以上电子书籍均可在 ftp://ibmtc@202.118.239.46/Document/Ebooks/Linux 找到

shell基础12篇

shell13问

[编辑]

任务

要求编写一个shell脚本，实现以下功能：

  1. 自动添加一个用户和密码，例如用户密码均为abcd
  2. 新建/home/abcd目录，将其属主设为abcd。
  3. 将/etc/shadow新生成的一行中的第一和第二段复制到/etc/exim/passwd中，例如

/etc/shadow中最后一行是abcd:$1
$fZEa7JlQ$oC5xMDGZT1ijtRnJrC.Mz/:13036:0: 99999: 7:::
现要在/etc/exim/passwd中添加一行： abcd:$1
$fZEa7JlQ$oC5xMDGZT1ijtRnJrC.Mz/


如何删除一个文件夹及其子目录？
如何在vi的命令模式中插入内容?
如何快速观看开机的硬件检测
如何察看所有的进程
如何终止一个进程，假设只知道进程名，或只知道进程号
如何察看一个目录中的所有文件（包括隐藏文件）

2.熟悉GCC、GDB

[编辑]

主要内容

掌握GCC的全过程，包括vi编辑、编译成汇编代码、中间代码和可执行文件。了解GDB的调试技巧。初步了解Makefile，可以自己写简单的Makefile文件。

[编辑]

推荐书目

  • （经典-必读）提问的智慧
  • 《Linux从入门到精通》
  • GCC编译与链接
  • Linux中x86的内联汇编
  • Linux 汇编语言开发指南
  • 缓冲区溢出&GCC内存分配
  • 缓冲区溢出&GCC内存分配
  • 表示高级话题，有能力的同学可以看看
[编辑]

时间

一周: 7.19-7.25

[编辑]

任务

编写一个程序，设有一个结构体数组： 

#define MAXSIZE 10
struct entity
{
   char* name;
   int value;
};
entity* table[MAXSIZE];

表中的值自己设，但是要随机。 要求：

  1. 对表中各项进行排序
  2. 将表存入文件
  3. 先建一个程序，能读出该表。
  4. 使用GCC生成中间文件、汇编文件和可执行文件；能使用GDB进行调试：掌握单步、断点调试

提示：存入文件的方式自己设计。有能力的同学可以尝试一下进行类型转化，将entity转换为char*，然后以二进制的方式进行存放。

[编辑]

3.了解网络管理

[编辑]

主要内容

了解基本的网络知识，熟悉各种网络服务的配置。

了解ftp、apache、mysql、php、tomcat的配置。如果有能力可以了解mail、bugzilla、cvs等配置[可选]。

[编辑]

任务

ftp的配置(可以选用vsftp、proftp)，配置要求：

  1. 不依赖于xinitd，作为一个单独的服务
  2. 允许匿名访问。
  3. 设置一个incoming目录,用户可以上传文件。
  4. 一个pub目录，用户为只读。有一个admin用户，能够对pub目录进行增删。
  5. 设置一个staff目录，用户a可以访问，匿名用户不可以访问。
  6. （可选）设置目录staff为可浏览属性，非a用户不可以浏览，即不可见。
  7. 设置主机限制，屏蔽173.26.100.84对ftp的访问（测试的时候，换一个别人的ip）。

bind的配置：

  1. 实现域名解析的功能即可。
  2. 将www.ibmtc.org的ip解析为202.118.250.18
  3. 将www.hippo.org的域名解析为173.26.100.59

Apache配置：

  1. 从源码安装，编译时加入php、mysql等模块。
  2. 配置虚拟服务器，实现在Apache上开启多个服务，通过不同端口绑定。
  3. 在自己的机器上配置wordpress。

任务

尽量减少相关工具的使用，采用手工正则表达式实现。当然，不管白猫黑猫，抓住老鼠就是好猫！

  • 到工大网络中心匹配出ip列表。 
格式如下：
58.17.0.0                0.0.255.255    255.255.0.0
需要得到：58.17.0.0/255.255.0.0，并将所得到的ip列表保存到一个文件里面。
提示：可以使用elinks、html2text以及grep、sed、awk
[可选]不使用elinks、html2text等html处理工具，直接匹配html文件（注意html中的标签）。
  • 到问天网上分析天气查询的协议 
查找哈尔滨、北京、上海的天气，剔除所有标签，保留天气信息。 
写成一个查询天气的脚本。
  • 登录教务处，查询自己的成绩。抓取页面，分析页面格式，提取成绩信息。 
按照原来的格式输出到一个文件中。
格式如下：
课程号 课程名 授课教师 学期 类别 学分 学时 平时 期中 期末 总成绩 备注 成绩复核

提示：按照页面中的table标签进行数据的过滤。

5.编译内核

为自己的机器定制一个内核，并尽量的进行裁减，使内核的大小尽量的最小。

  • 尝试使用不同的发行版编译，Debian系与RedHat系
[编辑]

参考资料

《Linux In A Nutshell》 该书可以从46上下载

[编辑]

时间

三天 8.10-8.13

[编辑]

实验

[编辑]

实验的准备

  1. 内核版本：2.6.20(根据内核的发展逐步确定版本)
  2. 基本编译工具：gcc make ld（相应版本根据不同的发行版确定）
  3. 其他工具：ccache distcc arm-linux-cc
  4. 编译主机：选用三台主机，每台主机均安装Ubuntu（Debian） Redhat SuSe(Redhat和SuSe建议采用基本安装，不要添加过多的服务)
[编辑]

实验目的

本此实验主要为了让大家熟悉内核编译的过程以及一些相关的技巧，并且通过实验了解内核编译所耗费的时间

[编辑]

实验内容

  • 主机的配置
  • 内核版本
  • 生成内核的大小
  • 编译内核耗费的时间

Part 1

  1. 使用 make allnoconfig 的方式进行编译
  2. 使用 make defconfig 的方式进行编译
  3. 使用 make allmodconfig 的方式进行编译
  4. 使用 make allyesconfig 的方式进行编译
  5. 根据主机配置，进行内核编译

Part 2

  1. 安装配置好distcc的环境，可以进行分布式编译
  2. 使用make CC=distcc gcc 做为参数，按顺序重复Part 1的1-4实验

Part 3

  1. 使用make CC=ccache gcc 做为参数，按顺序重复以上实验(注意在每次实验之后不要用make mrproper清空配置文件)
  2. 使用make -j NN 做为参数，按顺序重复以上实验(注意在每次实验之后不要用make mrproper清空配置文件；注：多核处理器使用效果比较好)
[编辑]

实验报告

请使用OpenOffice完成实验报告的撰写工作，相关数据请采用表格进行组织。

[编辑]

选作实验
