第二章修改意见

Wikipedia，自由的百科全书

[编辑]

谢煜波

第二章 p23第二段最后一行，“作为它基本参数”把“它”字去掉要通顺一些 第三段，第二行，原文中讲子进程关闭文件，只是当做一个例子，主要是说：“一个进程对文件表的更改不会影响到其它进程”，这个更改显然不仅仅是关闭文件，但是译文就恰恰表现出了这样的意思。 倒数第二段，标准错误译为标准错误输出比较符合一般翻译习惯。另外，预处理器宏比较少见，就说预处理宏就可以了 最后一段，“用于对于”重复了，去掉“对于”要通顺一些

2.1.1 The open() System Call在代码后面还h有一段话怎么没了？

2.1.1.1 “open(标志)”译为“open()的flags参数”要好一些 第二行，“用于只读。。”，“于”字去掉 示例代码不对，引号看起来很奇怪，“&=&” 这是什么？ 代码下面第二段：“标志参数”改为“flags“

25页 O_DIRECTORY，是opendir()在自己内部使用此参数，不是opendir()本身是内部使用的。译文把意思表达反了

O_LARGEFILE，我印象当中32位的机器上也可以打开64位偏移量的文件，sf就是这样的。请再确认一下此标志是否真的只能用于64位机器。


O_NONBLOCK，“用于定义FIFOs”，把“定义”去掉，不是用它们来定义FIFOs的。另外文中的FIFOs建议都直接译为FIFO，按中文习惯，名词后面一般不加s。

O_SYNC，翻译有误。原文是说当数据被真正写到磁盘之前此操作不会完成，也就是说调用不会返回，并不是在说不允许任何写操作。

O_TRUNC，翻译有误，O_TRUNC标志本身并不表示是否允许写操作。原文实际的意思是，在写标志存在的情况下，此标志。。。。

2.2 第二段第一句“读去文件”应是“读取文件” 第三段第一句，原文是"read up to”，所以最好翻译为“至多读len个字节" 第三段第二行最后一句话翻译不准确，原文是“The file position is advanced by the number of bytes”，是说“文件位置将会增加若干字节”，译文没译出这个意思。 2.2.1头上那段代码多了一个:号，并且这段代码后原文还有一段，译掉了


2.2.1 第二段第二行，原文中有“用o来表示EOF”的意思，但是译文把这个译没了。另外，建议保留原文EOF就是end of file的说明。 第二段倒数第三行，“这与EOF时的返回不同”应是“这与返回EOF时不同” 第四段那些不同的结果应当是一个列表，列表一般用黑点，译文中用的是“~”，用这种符号表示列表项极其少见。 列表第一项：“所有len个读取字节”，应为“所有len个被读取的字节” 第二项第二行最后，“有效读入字节”，原文是"available”，译为“可被读入字节”，或去掉“读入”，直接译为“有效字节”更通顺一些。最后一句“buf的空闲空间”，译为“buf的剩余空间”更好 32页第三行，“而读请求应该在之后重开”，这里“重开”很难懂

2.2.2 32页代码下面一段，“buf增加相应数量”不是很好理解

2.2.3 第一段倒数第二行，“进行I/O”译为“进行I/O操作”？ 第二段第二行，“O_NONBLOCK在open中给出”不如译为“在open()时指定O_NONBLOCK参数" 第二段最后一行，原文中没有“或者”的意思，原文意思是“否则将可能因为数据丢失而导致严重错误”，原文认为数据丢失是出现错误的原因，而不是与出现错误并列。

34页第一行，“或者不是用来读的打开文件”－－>“或者不是以可读的方式打开的”？

2.1.2 第二段第三行，原文中有个用括号括起来的setgid，建议在译文中保留，这样读者知道啥叫“设置组ID位”

第29页，最下面代码上面的一句话，“即使在用户空间。。。”不通

2.1.5 最后一句，“提示用户换个文件”改成“提示用户换个文件名”

2.2 第二段第一句“读去文件”应是“读取文件” 第三段第一句，原文是"read up to”，所以最好翻译为“至多读len个字节" 第三段第二行最后一句话翻译不准确，原文是“The file position is advanced by the number of bytes”，是说“文件位置将会增加若干字节”，译文没译出这个意思。 2.2.1头上那段代码多了一个:号，并且这段代码后原文还有一段，译掉了


2.2.1 第二段第二行，原文中有“用o来表示EOF”的意思，但是译文把这个译没了。另外，建议保留原文EOF就是end of file的说明。 第二段倒数第三行，“这与EOF时的返回不同”应是“这与返回EOF时不同” 第四段那些不同的结果应当是一个列表，列表一般用黑点，译文中用的是“~”，用这种符号表示列表项极其少见。 列表第一项：“所有len个读取字节”，应为“所有len个被读取的字节” 第二项第二行最后，“有效读入字节”，原文是"available”，译为“可被读入字节”，或去掉“读入”，直接译为“有效字节”更通顺一些。最后一句“buf的空闲空间”，译为“buf的剩余空间”更好 32页第三行，“而读请求应该在之后重开”，这里“重开”很难懂

2.2.2 32页代码下面一段，“buf增加相应数量”不是很好理解

2.2.3 第一段倒数第二行，“进行I/O”译为“进行I/O操作”？ 第二段第二行，“O_NONBLOCK在open中给出”不如译为“在open()时指定O_NONBLOCK参数" 第二段最后一行，原文中没有“或者”的意思，原文意思是“否则将可能因为数据丢失而导致严重错误”，原文认为数据丢失是出现错误的原因，而不是与出现错误并列。

34页第一行，“或者不是用来读的打开文件”－－>“或者不是以可读的方式打开的”？

2.3 第二段，“当前文件位置开始”－－》“当前位置开始”，“写入到文件中”－－》“写入文件中” 第四段，“像read()”－－》“像read()”一样

2.3.2 第一段，“通过O_APPEND”－－》“通过指定参数O_APPEND”。“而是”前面的句号应为逗号 第二段第三行，“而将指向。。”前面的句号应为逗号。“第二个进程写入的大小”－－》“第二个进程写入的字节数”？ 第二段倒数第二行，翻译有误，原文是说“如果不进行显示的同步操作，多个进程就不能进行追加写”，不是不能相互同步地进行追加写操作。 第三段第二行，“即便是多个写者”－－》“即便有多个写者” 第三段第二行，原文有更新文件位置是在写请求之前的意思，译文没有这个意思，反倒让人觉得是写操作完成后带来的的位置更新 第三段第四行，原文只是说"因为位置更新是自动完成的",译文译成了“因为位置更新是原子操作”

2.3.3 第一段，“通过O_NONBLOCK”－－》“通过设置O_NONBLOCK参数”

2.3.4 错误代码的排版极乱

EBADF，“或者不能用来进行写操作”，最好译为“或者不是用可写方式打开的”，因为不能进行写操作的错误代码应当是EINVAL

2.3.6 第二段，“然而”不好，这里没有需要转折的语义 第二段倒数第三行，“一种叫做回写的过程”－－》“此过程称为写回”？ 第三段,"不会改变POsIX语义”－－》“不会改变POSIX定义的语义”? 第四段，倒数第二行，“而最终所有缓冲区将回写都没有问题”，相当拗口。原文的意思其实是“这只在系统崩溃时才是问题，因为在一般情况下，最终所有的缓冲区都将正确的写回” 第五段第二行，“就是说”，原文是“say”，感觉这里的“say”翻译成“比方说”要好一些 第五段倒数第二行，“多进程可能会污染到在一片缓冲区。。”，“在一片”改为“同一片”，另外，“污染”直接译为"写”或“更新”是不是要好一些？

38页， 第二段第一行，“以一种适时的方式写入”－－－》“适时写入”？ 第二段第二行，“最大缓存时效”－－》“缓存最大时效” 第三段第一行，“强制给出的文件缓存回写”拗口，不如直译为“强制文件缓存写回”，“写同步”－－？“写操作同步”？ 第三段第二行，“下一段讨论的主题”－－》“下一节讨论的主题”

2.4 第三行，“控制数据写入磁盘的时间”改为“控制数据被写入磁盘的时间”，缺少“被”字，容易理解为控制的是写入数据总共需要花的时间。

2.4.1 第二段第一行，“文件描述符fd必须是打开的”，原文强调了必须是以写的方式打开的

39页 第一段，”除了刷新数据"可以去掉，因为后面已经有个“区别在于。。。”，放在这里拗口

2.4.2 EIO，“此类错误经常在错误发生处被捕获”，我觉得原文的意思是“此类错误常常需要在此处理”或者“此类错误常常需要立即处理” 下面一段第一句，翻译有误，原文意思是“即使实现了fdatasync()”，译文没有即使的意思

第三段第二行，“初始化”－－》“开始”，“启动”？

2.4.4 那段代码O_SYNC多了一个\

42页 第一行，原文的意思是因为O_SYNC要求更强，所以性能会有损失，译文译为“对O_SYNC有高要求”意思表达不是很清晰
