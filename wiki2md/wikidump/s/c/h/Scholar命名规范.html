1．持久化对象

存放位置：

net.cowoo.skeleton.domain

命名规则：

注意与数据库中的表名保持一致

属性及方法：

属性的与表格中的域保持一致，但是采用java属性的命名习惯，与数据库中的下划线分离相区别

举例：

对应于数据库中的groups表，JavaBean的名称为Group，表中的creator_id域对应的属性命名为creatorId

2．DAO

1) DAO的接口

存放位置：

net.cowoo.skeleton.dao

命名规范：

在对应的JavaBean名称之后添加Dao

属性及方法：

DAO接口一般包括以下几个方法： 

在数据库表中获得一个条目：getXxx方法，用id作为参数，返回一个对象

在数据库表中获得多个条目：getXxxs方法，无参数，返回一个List
在数据库表中添加一个条目：saveXxx方法，用对象作为参数，无返回值
在数据库表中删除一个条目：removeXxx方法，用id作为参数，无返回值
在数据库表中更新一个条目：updateXxx方法，用对象作为参数，无返回值
根据某个属性获得一个条目：getXxxByYyy方法，用属性为Yyy作参数，返回对象
根据某个属性获得多个条目：getXxxsByYyy方法，用属性为Yyy作参数，返回List

举例：

对应于Group，DAO接口的名称为GroupDao，在接口中有下列方法： 

     Group getGroup（String id）；
     List getGroups（）；
     Void saveGroup（Group group）；
     Void removeGroup（String id）；
     Void UpdateGroup（Group group）；
     List getGroupsByName（String name）；

2) DAO的具体实现

存放位置：

net.cowoo.skeleton.dao.hibernate

命名规范：

在对应的DAO名称之后添加Hibernate

属性及方法：

实现对应DAO接口中定义的方法，因为继承自BaseDaoHibernate，get、save和remove的基本操作可以调用super的方法

举例：

对应于GroupDao，DAO接口实现的名称为GroupDaoHibernate

3．Manager

1) Manager的接口

存放位置：

net.cowoo.skeleton.service

命名规范：

“I” + “对应的JavaBean名称” + “Manager”

属性及方法：

Manager接口一般包括Dao层次中的基本方法，除此之外又定义了一些复杂的数据库操作方法。 举例：对应于Group，Manager接口的名称为IGroupManager

2) Manager的具体实现

存放位置：

net.cowoo.skeleton.service.impl

命名规范：

“对应的JavaBean名称” + “Manager”

属性及方法：

i. 拥有一个处于Manager层底层的DAO属性，通过它定义基本的或者更复杂的数据库操作

ii. 实现对应Manager接口中定义的方法

举例：

对应于Group，Manager的实现的名称为GroupManager

4．Action

存放位置：

net.cowoo.skeleton.web

命名规则：

“相关的功能名称” + “Action”

属性及方法：

i. 拥有一个Manger属性，用来处理持久化的工作，属性名为Manager类名首字母小写；

ii. Action中一般包括多个方法，一些常用的方法命名（用于管理员模式）统一如下： 

对应于实体表的Action：
区分用户使用和管理员使用的原因是跳转页面的不同
用户使用的：
创建：使用“createXXX”
编辑：使用“editXXX” (跳转到编辑的页面)
更新：使用“updateXXX”  (实际的数据库更新动作)
删除：使用“removeXXX”
管理员使用的：
存储一个条目：使用“save”
编辑一个条目：使用“edit”
更新一个条目：使用“update”
删除一个条目：使用“remove”
显示单个条目：使用“item”
显示所有条目：使用“list”

iii. Action中的每个动作都有相应的页面，每个页面单设一个属性，以便依赖注入 如果是信息提交页面交给Action处理，成功之后的页面，对应的属性名为 “方法名” + “SuccessPage”；失败之后的页面，对应的属性名全部为“errorPage”


举例：

GroupAction的方法有create，edit，viewItem，viewList,list；属性有groupManager，createSuccessPage，editSuccessPage，viewItemSuccessPage，viewListSuccessPage，listSuccessPage,errorPage等等。

5．页面的命名

存放位置：

web\jsp\是jsp页面的根目录，某一功能模块的页面放在根目录的相应文件夹中

命名规则：

文件夹的名称与功能相对应就可以，页面名称的单词之间用大写字母分离，不要再后面加上 “Page”，这样比较简洁

举例：

群组有关的功能放在web\jsp\groups目录中，目录下有create、createSuccess、edit、editSuccess、viewItem 、viewList、error等等。

6．方法的参数

命名规则：

方法的参数和类中唯一的属性尽量不要用简写的形式，建议使用将类名的首字母小写的形式

举例：

Action类中的GroupManager命名为groupManager
