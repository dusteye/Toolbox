19管理方案

[编辑]

以root身份登陆19，转到/home/cowoo目录

cd /home/cowoo/
[编辑]

更新ScholarLink代码

rm -rf ScholarLink_2007
cvs checkout ScholarLink_2007
[编辑]

编译ScholarLink代码

cd ScholarLink_2007
ant build
[编辑]

将编译后的代码发布到服务器

rm /var/www/tomcat-instances/ScholarLink/web -rf
cp -R web /var/www/tomcat-instances/ScholarLink/
[编辑]

重启服务器

cd /var/www/tomcat-instances/ScholarLink
./shutdown.sh
./startup.sh

SL与19配置ScholarLink实例的异同

19上的配置方案是以用户定制(临时指定$CATALINA_BASE)的方式配置的，即在应用下建conf，logs，work等目录，
然后在conf/server.xml下指定docBase，这个docBase一般来说下面会有WEB-INF文件夹，便如web或WebRoot这
种目录。
而SL服务器上是典型的tomcat下实例配置方式，即在$CATALINA_HOME/webapps将整个应用拷贝过去。
理解了这个就不难分辩SL服务器和19部署的不同之处:
#build.properties中若在SL服务器部署得改为/usr/local/tomcat5/webapps/ScholarLink
