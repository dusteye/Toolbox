方法规范

1、Action的方法应该采用动词或者动宾结构，因为是URL的一部分，应该保持在一个到连个单词为宜。

2、常用的增删查改采用add,remove,get,update,show,list这几个动词。

3、有时候有一些准备的页面，比如add之前填写的表单的页面，添加pre前缀，比如preAdd、preUpdate等等。

先定这三条吧，感觉方法不需要太多的规定。 

=== ClassName ===
  新名：NewCalssName
        新名要求保留action尾缀，删除和包名含义重复的前缀，在保证可读性的基础上尽量减小单词长度
  标记：[分割|删除|保留]
        分割：将原action方法进行分割，需在备注里写明分割涉及到的方法、分割理由、分割去向
        删除：最终将此类删除
        保留：最终保留此类，并在备注里说明分配给这个类的url
  ID  ：xml配置文件中使用的bean id
  作者：使用~~~~进行署名
  URL ：分配给这个类的url，例如：/conf/admin/chair/ 命名原则是将web开始的包名+类名中的点换成/
  备注：需要详细说明重构方法
