URL

核心：仿照RoR，package+actionClassName+MethodName ==> URL 

方案：从web package开始，包名/[包名...]/类名/方法名.htm

实现目标： DONE

  1. 对于特殊情况，允许在xml中进行原有形式的配置，并且优先级高于自动获取。
  2. 自动获取情况下
    1. 根据request url，SLUrlHandlerMapping分析出对应的action id，注册并返回handler。
    2. 根据request url, SLMethodNameResolver分析出methodName。 
例子：                   http://202.118.250.19:8084/conference/page.htm?method=viewList&id=6
按照之前的URL规范重写为：http://202.118.250.19:8084/conf/page/viewList.htm?id=6
去掉domain部分后的URL为：/conf/page/viewList.htm?id=6
SLUrlHandlerMapping分析得出的bean id == confPage
SLMethodNameResolver分析出methodName为：viewList

参考资料： [1]

方法规范

1、Action的方法应该采用动词或者动宾结构，因为是URL的一部分，应该保持在一个到连个单词为宜。
2、常用的增删查改采用add,remove,get,update,show,list这几个动词。
3、有时候有一些准备的页面，比如add之前填写的表单的页面，添加pre前缀，比如preAdd、preUpdate等等。

先定这三条吧，感觉方法不需要太多的规定。

JSP

/jsp/结构：

  1. jsp内的结构参照URL来设置，每一个action类有自己一个单独的文件夹，一个方法对应至少一个页面。
  2. jsp内所有文件夹名称采用小写字母，并尽量减少长度，原则是前面文件夹路径携带了的信息就不要包含在文件名里了。jsp文件名称主要和method保持一致。
