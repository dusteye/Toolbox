测试代码范例

package org.scholar.skeleton.service;
import static org.easymock.EasyMock.*;
import org.scholar.skeleton.dao.UserDao;
import org.scholar.skeleton.domain.User;
import org.scholar.skeleton.service.impl.UserManager;
public class UserManagerTest extends BaseManagerTestCase {
	private UserManager userManager;
	private UserDao mock;
	protected void setUp() throws Exception {
		mock = createMock(UserDao.class);     //建立mock对象
		userManager = new UserManager();      //初始化被测试对象
		userManager.setDao(mock);             //将mock对象通过set方法加入到被测试对象中
	}
	public void testGetUser() throws Exception {	
		User testUser = new User();    //返回值初始化
		testUser.setId("1");
		testUser.setEmail("liushi@cs.hit.edu.cn");
		testUser.setNickName("liushi@cs.hit.edu.cn"); 	
	
		reset(mock);          //复位mock
		expect(mock.getUser("1")).andReturn(testUser);   //录制mock对象行为
		replay(mock);         //停止录制，进入回放
		User user = userManager.getUser("1");            //调用被测试类待测方法
		verify(mock);         //检查录制方法是否已合法调用	
	
		assertEquals("get", testUser, user);  //用assert检查返回值是否为预期结果	
	}
	public void testSaveUser() throws Exception {
		User testUser = new User();
		testUser.setNickName("cowoo");
		testUser.setEmail("cowoo@hit.edu.cn"); 
	
		reset(mock);
		mock.saveUser(testUser);
		replay(mock);
		userManager.saveUser(testUser);
		verify(mock);  //没有返回值的方法只需测试mock对象的方法是否按照预期调用
	}
	public void testRemoveUser() throws Exception {
		reset(mock);
		mock.removeUser("1");
		replay(mock);
		userManager.removeUser("1");
		verify(mock);
	}
	public void testGetUserByEmail() throws Exception {
		String email = "liushi@cs.hit.edu.cn";
		User testUser = new User();
		testUser.setEmail(email);
	
		reset(mock);
		expect(mock.getUserByEmail(email)).andReturn(testUser);
		replay(mock);
		User user = userManager.getUserByEmail(email);
		verify(mock);
	
		assertEquals(testUser , user);		
	}
}

格式规范

test方法内部分为三个部分：测试数据初始化、mock段、assert段。

第一部分是用于入口参数及mock的制定返回值的初始化。

第二部分是用于easymock的测试过程，分为reset()、mock、replay()、Manager、verify()五个阶段。

第三部分是用于利用assert对返回值进行验证，包括NotNull验证、必要的参数验证。

第一、第三部分可视实际情况增添，第二部分是必须要有的。每部分之间用空行隔开，部分内部不建议使用空行。

[编辑]

几点说明

  1. 返回值 在测试方法最开始进行的返回值初始化中，这个返回值是mock对象（即被测试类的内部成员）返回给调用者（即被测试类）的，与被测试方法的返回值没有必然的相等关系。只不过我们的类中Manager是直接返回Dao的返回值，所以在测试的时候，被测试方法返回值就是我们让mock对象返回的值。
  2. 最后决定Action不用easymock来测试，而是只用JUnit，应为如果用easymock来测得话，有两点不好的地方 
 一、有些方法会调用其他方法，产生不必要的mock操作，扰乱测试目的。
 二、需要准备的test数据太多，使得测试代码比被测试类的都多。

代码范例

package org.scholar.skeleton.web;
	
import org.springframework.mock.web.MockHttpServletRequest;
import org.springframework.mock.web.MockHttpServletResponse;
import org.springframework.web.servlet.ModelAndView;
	
public class UserActionTest extends BaseWebTestCase {
	
	private UserAction userAction = null;
	
	public void setUserAction(UserAction userAction) {
		this.userAction = userAction;
	}
	
	public void testAdd() throws Exception {
		MockHttpServletRequest request = new MockHttpServletRequest();
		MockHttpServletResponse response = new MockHttpServletResponse();
		request.setMethod("POST");
		request.addParameter("nickName","cowoo");
		request.addParameter("password","liushi");
		request.addParameter("email","liushi@hit.edu.cn");
	
		ModelAndView mv = userAction.add(request, response);
			
		assertNotNull(mv);
		assertNotNull(request.getAttribute("users"));
		assertEquals("admin page is ", "users/userAdmin", mv.getViewName());
	}
		
	public void testRemove() throws Exception{
		MockHttpServletRequest request = new MockHttpServletRequest();
		MockHttpServletResponse response = new MockHttpServletResponse();
		request.addParameter("id" , "1"); 
	
		ModelAndView mv = userAction.remove(request , response);
			
		assertNotNull(mv);
		assertEquals("admin page is ", "users/userAdmin", mv.getViewName());
			
	}
	
}

格式规范

test方法内部分为三个部分：MockHttpSerletXXX建立及初始化、Action方法调用、assert段。

第一部分是用于初始化request中的参数，以便于传入Action使用。

第二部分是Action的方法调用，一行足以。

第三部分是用于利用assert对返回值进行验证，包括mv的NotNull验证、requset中心添加的参数的NotNull验证、返回页面的正确性验证等。

每部分之间用空行隔开，部分内部不建议使用空行。 Request、Response对象同时建立，之后是对request进行添加Parameter的操作。

JsUnit 简介

我们的测试使用JsUnit框架，用JsUnit编写测试与JUnit很相似，相似点如下：

JUnit JsUnit
测试函数 不带任何函数，必须有一个前缀test 同JUnit相同
测试类 测试函数包含在testClass中 测试函数包含在testPage中
测试集 包含testClass和testSuite 包含testPage和其他testSuite
断言方法 assertXXX方法 基本与JUnit相同
  • JsUnit具体特性请参见《Ajax基础教程》
[编辑]

JsUnit 在我们项目中的使用方法

1.我们项目中配置的JsUnit在项目中的ScholarLink/jsunit文件夹下，在使用测试时在浏览器中(为了方便，可以用eclipse自带的浏览器)打开ScholarLink/jsunit/testRunner.html

2.在页面的浏览按钮中找到测试页,测试页统一放置在ScholarLink/web/jsp/jsUnitTest文件夹下

3.点击Run即可进行测试

[编辑]

JsUnit 测试页的写法

1.让测试页中包含JsUnit的核心测试代码,通过在页面中添加元素<script language="JavaScript" type="text/javascript" src="../../../jsunit/app/jsUnitCore.js"></script>来实现

2.让测试页中包含待测试代码,通过在页面中添加元素<script language="JavaScript" type="text/javascript" src="../scripts/testPageName.js"></script>,testPageName.js待测试JavaScript文件名

3.可以通过setUp() tearDown()方法创建测试环境，通过断言进行测试

4.还可以使用testSuite加载多个测试页
