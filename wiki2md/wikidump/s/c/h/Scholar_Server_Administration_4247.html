系统安装与配置

[编辑]

基本系统安装

  • 使用Debian Etch安装盘，安装操作系统，并更新安全补丁。
  • 安装Apache (apt-get install apache)
  • 安装Mysql 5.0(apt-get install mysql-server-5.0)
  • 安装phpmyadmin: (可选，主要用户基于Web的Mysql管理)
  • 安装postfix邮件服务器(apt-get instal postfix, 配置时选择Internet Site)
  • 安装JDK ：官方安装指南 。安装后需要设置环境变量，将以下加入到/etc/profile中 
 export JAVA_HOME=/usr/local/jdk1.6.0_02
 export JAVA_BIN=/usr/local/jdk1.6.0_o2/bin
 export PATH=$PATH:$JAVA_HOME/bin
 export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
  • 安装Tomcat: 将tomcat解压缩到/usr/local/tomcat。进入/usr/local/tomcat/bin 目录下，将所有的.sh文件设置为可执行。使用./startup.sh 即可启动tomcat服务。

下载mod_jk.so

直接下载binary就行，不用编译。扔到/usr/lib/apache2/modules/里面去。

[编辑]

配置mod_jk.conf（/etc/apache2/mod_jk.conf）

 JkWorkersFile /etc/apache2/workers.properties
 
 # Where to put jk logs
 JkLogFile /usr/local/tomcat5/logs/mod_jk.log
 
 # Set the jk log level [debug/error/info]
 JkLogLevel info
 
 # Select the log format
 JkLogStampFormat "[%a %b %d %H:%M:%S %Y]"
 
 # JkOptions indicate to send SSL KEY SIZE,
 JkOptions  +ForwardKeySize +ForwardURICompat -ForwardDirectories
 
 # JkRequestLogFormat set the request format
 JkRequestLogFormat "%w %V %T"
 #JkMount /servlet/*  worker1
 #JkMount /*.jsp worker1
 
 #for webwork
 #JkMount /*.action

我们没有在mod_jk.conf里面写JkMount，因为scholar-link只是这台主机上的一个域名，需要在那个虚拟主机上进行配置。

[编辑]

配置/etc/apache2/apache2.conf

在文件最后加上 

 LoadModule jk_module /usr/lib/apache2/modules/mod_jk.so
 Include /etc/apache2/mod_jk.conf

配置一个虚拟主机 

 <Virtualhost 219.147.187.186>
         ServerName www.scholar-link.com
         ServerAlias beta.scholar-link.cn
         ServerAdmin webmaster@localhost
 
         DocumentRoot /usr/local/tomcat5/webapps/ScholarLink/
 
         <Directory /usr/local/tomcat5/webapps/ScholarLink>
                 Options Indexes FollowSymLinks MultiViews
                 AllowOverride None
                Order allow,deny
                allow from all
        </Directory>
 
        JkMount /* worker1
 </Virtualhost>

注意到我把这个虚拟主机下的所有请求都JkMount到worker1去了，这是不必要的，有些静态文件，比如说图片、javascript以及css直接从apache走比较快，这个以后再改进。

我在sl服务器上(不是19)加了这么两句话: 

JkUnMount /jsp/images/* worker1
JkUnMount /jsp/scripts/* worker1


[编辑]

创建/etc/apache2/workers.properties

 # Defining a worker named worker1 and of type ajp13
 worker.list=worker1
 
 # Set properties for worker1
 worker.worker1.type=ajp13 
 worker.worker1.host=localhost 
 worker.worker1.port=8009
 #worker.worker1.lbfactor=50 
 #worker.worker1.cache_timeout=600 
 #worker.worker1.socket_keepalive=1 
 #worker.worker1.socket_timeout=300
[编辑]

配置/usr/local/tomcat5/conf/server.xml

Tomcat5.5.23中AJP1.3默认是开启的。 

  <Service name="Catalina">
 
     <Connector port="8080" maxHttpHeaderSize="8192"
                maxThreads="150" minSpareThreads="25" maxSpareThreads="75"
                enableLookups="false" redirectPort="8443" acceptCount="100"
                connectionTimeout="20000" disableUploadTimeout="true" />
 
     <Connector port="8009"
                enableLookups="false" redirectPort="8443" protocol="AJP/1.3" />
 
     <Engine name="Catalina" defaultHost="localhost" debug="0" >
 
         <Valve className="org.apache.catalina.valves.FastCommonAccessLogValve"
                  directory="logs"  prefix="localhost_access_log." suffix=".txt"
                  pattern="common" resolveHosts="false"/>
       <Host name="localhost" debug="0" appBase="webapps" unpackWARs="true" autoDeploy="true">
         <Alias>www.scholar-link.com</Alias>
         <Alias>beta.scholar-link.cn</Alias>
         <Context path="" docBase="ScholarLink" debug="1"/>
 
         <Valve className="org.apache.catalina.valves.FastCommonAccessLogValve"
                  directory="logs"  prefix="scholarlink_access_log." suffix=".txt"
                  pattern="common" resolveHosts="false"/>
       </Host>
     </Engine>
 
   </Service>

主要是配置Host，要有这些虚拟主机的名字。另外，defaultHost要用localhost，因为前面worker的host用的localhost，这两个好像要对应。

获取Scholar-Link 代码

  • 以root身份登陆服务器，转到/root/Scholar_Link目录 
cd /root/Scholar_Link
  • 更新ScholarLink代码 
rm -rf ScholarLink_2007
cvs checkout ScholarLink_2007
[编辑]

编译Scholar-Link代码

  • 修改LANG 
export LANG=en_US.UTF-8
然后echo $LANG一下看是否从C改成en_US.UTF-8了
这是因为在SL服务器上local编码为asiII,若JAVA文件中有中文,编译时会有警告,而当显示在页面时会有乱码
嗯,这个问题困扰了我好久好久
reference:
1
2
  • 修改build.properties 
由于SL服务器与19部署方式不同,故需将build.properties中deploy.path改为
/usr/local/tomcat5/webapps/ScholarLink
具体原因见Scholar-Server-Management
  • 编译ScholarLink代码 
cd ScholarLink_2007
ant build(这一步也会执行deply的动作,即把代码copy到deploy path目录下)
ant db-create(初次安装需要执行这个任务)
ant db-load_JunitTestData(初次安装需要执行这个任务)
ant deploy(将ScholarLink发布到指定的目录,此动作可省略,因为build中已经包含)

发布Scholar-Link代码

  • 将编译后的代码发布到服务器: ant build会自动将生成的代码发布到指定目录

cd /usr/local/tomcat5/bin 下
./shutdown.sh
./startup.sh
  • 访问www.scholar-link.com 确认是否正确发布情况
