Eclipse中安装SpringFramework IDE

本来以为不应该有什么问题，去spring官网下载-〉安装，就可以。但是去了才发现这个安装是用update安装，而且需要很多额外的jar支持，比如GEF（Graphical Editing Framework）、EMF（Eclipse Modeling Framework）、WTP（ Web Tools Platform，需要里面的WST，在这里堵住了）等。安装过程比较繁琐，中途还遇到更新网站链接失败，所以这条路没走通。

又去Spring官网，下载了WTP的wtp-all-in-one-sdk-R-1.0.2-200604280245-win32.zip，是一个配制好的eclipse3.1，权且是解决了WTP的问题，在这个基础上升级安装Spring，很简单就搞定了（过程网上很容易搜到，就是Help-〉Software Updates-〉Find and Install……）。 

Spring IDE安装说明

Spring IDE实际上就是一个用来验证Spring的配置文件是否出错的一个Eclipse插件，配合Eclipse的WTP使用也还是不错，只是问题也不少，特别是WTP的同步问题，简直是烦死人（也许是我用的不得法。。。）。如果在Windows下开发，建议使用MyEclipse（ibmtc@46上有），对Spring的支持非常不错，用起来也很不错，就不用装Spring IDE了。如果是在Linux下开发，那就装WTP吧，不过在Linux下怎样才是一个最好的开发流程，使用哪些工具，如何使用，还在探索中。
