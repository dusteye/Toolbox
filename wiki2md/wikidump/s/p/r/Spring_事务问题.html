Spring 事务问题

Wikipedia，自由的百科全书

在使用JUnit对更新对象进行测试的时候 

        public void testUpdateGroup(){

String newname = "cowoo's group";

Group group = dao.getGroup("1"); log.error(group); group.setName(newname); dao.updateGroup(group);

Group result = dao.getGroup("1"); log.error(result);

assertEquals("Group Name",newname,result.getName()); }

updateGroup方法未实现测试也能通过，发现两次getGroup所获得的对象是同一对象，猜测可能是Spring的事务的问题。


[编辑]

事务参数

Spring的org.springframework.transaction.interceptor.TransactionProxyFactoryBean类中有transactionAttributes这个属性，其形式是Properties的形式（即key=value的形式)。在acegi的配置文件中有 

 <prop key="save*">PROPAGATION_REQUIRED</prop>
 <prop key="remove*">PROPAGATION_REQUIRED</prop>
 <prop key="*">PROPAGATION_REQUIRED</prop>

三行，如果生效过后，注册的saveUser方法测试无法通过，错误为：
