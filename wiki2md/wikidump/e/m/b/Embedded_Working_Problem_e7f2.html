DirectFB --4月8日

目前，DirectFB提供的测试工具，如；dfbsummon,dfbinfo等，在个人主机系统上运行时，系统错误只有以上的错误，即： 

   DirectFB/FBDev: Panning display failed!     
        --> Invalid argument



     The problem is solved.The key point is to grub video like
     this: video=vesafb:ywrap,mtrr:3
     when loading the kernel
     (see /Documentation/fb/vesafb.txt)

对于Intelfb的设置，请看 /Documentation/fb/intelfb.txt. 然而，在intel的主板上，以上的解决方案还有问题： 我将参数改成： 

      video=intelfb:1024x768-32@70,vram=8,mtrr=3

但是系统依然有错误： 在intel主板上，dmesg信息如下：

而在U盘系统中，以上的测试工具，能够正常的启动，加载DirectFB提供的库文件，但是，一旦屏幕刷新，显示就有问题。 刷新以后系统只显示黑色的屏幕，所有操作都不起作用，系统还无法恢复到原来的正常工作界面下，即Shell界面。 个人已经作的测试：修改库文件的名称，使工具无法加载，这时候系统在加载过程中就出现故障。这说明，在U盘的系统中 加载DirectFB是没有问题的；而系统的内核和我现在用的个人主机的内核一样，所以，也可排除内核有问题的情况，因此， 我认为出现问题应该在文件系统上，那么如何查找问题？问题的关键在哪？还在摸索中！！！
