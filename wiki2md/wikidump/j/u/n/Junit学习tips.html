Junit学习tips

Wikipedia，自由的百科全书

Junit是Java的经典测试框架，也是开发过程中必须的测试手段，从今天起我将陆续将自己学习和实践Junit过程中遇到的问题写下来。也许很多只是细节的小问题，但是累积起来还是很有参考价值的。也希望大家共同学习并分享自己的收获。

  • 配置Junit时会把junit.jar添加到环境变量中。在使用MyEclipseIDE创建一个Junit的工程后，会自动导入位于MyEclipse\plugins\org.junit_3.8.1\junit.jar,造成版本的差别，在我的例子中这种情况没有导致错误，从理论上说也不应该出错，不过有时候版本的问题会带来一些说不清道不明的小问题，所以建议大家还是别怕麻烦，重新导入Junit安装目录下的最新版本的junit.jar。：）
  • To create our own simple test case, we need to follow these steps: 
  1. Create an instance of junit.framework.TestCase.
  2. Define tests that return void and whose name begins with the string "test" such as testWasTransactionSuccessful(), testShow(), etc.).
  
  今天我写的例子中把testHelloWorld方法中的test写成了Test，结果出现了junit.framework.AssertionFailedError: No tests found in TestThatWeGetHelloWorldPrompt的错误提示。
