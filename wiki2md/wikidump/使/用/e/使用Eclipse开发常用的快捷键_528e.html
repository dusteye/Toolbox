使用Eclipse开发常用的快捷键

Wikipedia，自由的百科全书

目前Android的开发过程，常用Eclipse作为IDE。

而利用好Eclipse的快捷功能，通常能够到达加快开发效率的目的。

以下列写我常使用的Eclipse快捷键

  1. Ctrl + / : 注释
  2. Alt + /  : 自动补全
  3. Ctrl + Shift + O(ou) : 全局增添或者删除import的包
  4. Ctrl + Shift + F ：整理格式
  5. Ctrl + M : 编辑区域最大化
  6. Ctrl + O : 列出类中继承结构
  7. Alt + Shift + S : 列出编码常用选项
  8. Ctrl + F11 : 运行
  9. Ctrl + H : 查找
  10. 存盘 Ctrl+s(肯定知道)
  11. 快速修复 Ctrl+1
  12. 切换窗口 Ctrl+f6 <可改为ctrl+tab方便>
  13. ctrl+shift+M 导入未引用的包
  14. ctrl+w 关闭单个窗口
  15. F3 跳转到类、变量的声明
  16. F11 运行上次程序
  17. Ctrl + F11 调试上次程序
  18. Alt + 回下一个编辑点
  19. ctrl+shift+T 查找工程中的类
  20. Ctrl+D: 删除当前行
  21. Ctrl+Alt+↓ 复制当前行到下一行(复制增加)
  22. Ctrl+Alt+↑ 复制当前行到上一行(复制增加)
  23. Alt+↓ 当前行和下面一行交互位置(特别实用,可以省去先剪切,再粘贴了)
  24. Alt+↑ 当前行和上面一行交互位置(同上)
  25. Alt+← 前一个编辑的页面
  26. Alt+→ 下一个编辑的页面(当然是针对上面那条来说了)
  27. Alt+Enter 显示当前选择资源(工程,or 文件 or文件)的属性
  28. Shift+Enter 在当前行的下一行插入空行(这时鼠标可以在当前行的任一位置,不一定是最后)
  29. Shift+Ctrl+Enter 在当前行插入空行(原理同上条)
  30. Ctrl+Q 定位到最后编辑的地方
  31. Ctrl+L 定位在某行 (对于程序超过100的人就有福音了)
  32. Ctrl+K 参照选中的Word快速定位到下一个
  33. Ctrl+E 快速显示当前Editer的下拉列表(如果当前页面没有显示的用黑体表示)
  34. Ctrl+/(小键盘) 折叠当前类中的所有代码
  35. Ctrl+×(小键盘) 展开当前类中的所有代码
  36. Ctrl+Shift+E 显示管理当前打开的所有的View的管理器(可以选择关闭,激活等操作)
  37. Ctrl+J 正向增量查找(按下Ctrl+J后,你所输入的每个字母编辑器都提供快速匹配定位到某个单词,如果没有,则在stutes line中显示没有找到了,查一个单词时,特别实用,这个功能Idea两年前就有了)
  38. Ctrl+Shift+J 反向增量查找(和上条相同,只不过是从后往前查)
  39. Ctrl+Shift+F4 关闭所有打开的Editer
  40. Ctrl+Shift+X 把当前选中的文本全部变为小写
  41. Ctrl+Shift+Y 把当前选中的文本全部变为小写
  42. Ctrl+Shift+F 格式化当前代码
  43. Ctrl+Shift+P 定位到对于的匹配符(譬如{}) (从前面定位后面时,光标要在匹配符里面,后面到前面,则反之)
  44. 下面的快捷键是重构里面常用的,本人就自己喜欢且常用的整理一下(注:一般重构的快捷键都是Alt+Shift开头的了)
  45. Alt+Shift+R 重命名 (是我自己最爱用的一个了,尤其是变量和类的Rename,比手工方法能节省很多劳动力)
  46. Alt+Shift+M 抽取方法 (这是重构里面最常用的方法之一了,尤其是对一大堆泥团代码有用)
  47. Alt+Shift+C 修改函数结构(比较实用,有N个函数调用了这个方法,修改一次搞定)
  48. Alt+Shift+L 抽取本地变量( 可以直接把一些魔法数字和字符串抽取成一个变量,尤其是多处调用的时候)
  49. Alt+Shift+F 把Class中的local变量变为field变量 (比较实用的功能)
  50. Alt+Shift+I 合并变量(可能这样说有点不妥Inline)
  51. Alt+Shift+V 移动函数和变量(不怎么常用)
  52. Alt+Shift+Z 重构的后悔药(Undo)
  53. Alt+Shift+J 方法注释 ( Mac OS: Command+ Option+J)
