序

当Linux的内核开发人员感到恼怒的时候，他们总爱抛出这样的一句话：“用户空间仅仅是内核加载测试。”（还是没明白这句话的意思）


内核开发人员嘟囔着这句话的同时，总期待着能够尽可能的推卸掉所有用户空间代码运行失败的责任。在他们看来，用户空间开发人员应当回过头去看看他们自己的代码，任何问题都显然不是内核的过错。


为了证明这些问题通常不是内核导致的，一个Linux内核主要开发人员不辞辛劳的讲述着关于”为什么用户空间让人心碎“的讲座，他举了许多活生生的例子，而这些例子中的代码都是大家每天都赖以运行的恐怖的用户空间代码。其他的内核开发人员也已经创建了很多的工具来展示用户空间程序是怎样的滥用硬件资源和耗尽的笔记本电池。


但是，当内核开发人员在嘲笑用户空间代码仅仅是一个加载测试时，具有讽刺意味的是，我们发现所有的这些内核开发人员每天都依赖着用户空间的代码。否则，所有的内核将只会在屏幕上打印着状如ABABAB交替出现的模式的字符串。


今天，纵览世界的现已构建的操作系统，Linux是最灵活和健壮的，它能运行在所有的设备上，小至手机和嵌入式设备，大至世界上前500个超级计算机中的70%。没有任何其他的操作系统能够有如此好的考平台性和满足设备类型和环境的区别带来的挑战。


同时，伴随着内核的发展，运行在Linux用户空间的代码同样能够在这些平台上稳定的运行，这为人们提供了赖以生存的应用程序和工具。


在这本书中，Robet Love承担了一项令人项目的工作，那就是教导读者关于几乎所有Linux上的系统调用相关的内容。通过这种方式，他希望一方面能够让读者从用户空间的角度了解Linux内核是如何工作的，同时也掌握和利用这个强大的系统。


这本书将告诉你如何写出能够在各种Linux发行版本和硬件类型上的运行代码。它将让你理解Linux是如何运行的和掌握怎样才能充分的利用它的灵活性。


最后，这本书将指导你怎么写出不会崩溃的代码，而这是最美好的。

---Greg Kroah-Hartman

前言：

这是一本关于系统编程，特别是关于Linux的系统编程的书。系统编程是一项系统软件编写的实践工作，编写的代码运行在系统的底层，与内核和核心系统库进行直接对话。换一种说法就是，这本书的主题是关于Linux系统调用和其他底层函数，例如如有C库定义的函数。

虽然有很多涵盖了Unix系统的系统编程书籍，但专注于Linux的却很少，特别是关于最大的Linux发布版本和高级的纯Linux的接口就更少了。除此之外，这本书得益于长期深入对Linux的接触：我已经为Linux写了很多代码，有内核和编译于内核之上的系统程序。事实上，我已经实现了在这本书中所涉及的一些系统调用和提及的特征。因此，这本书包含了很多内部的知识，有关于系统接口表面上应当如何工作，还有实际上如何运行的知识，以及作为程序员如何才能高效的使用它们。这本书综合了一个独立成体的Linux系统编程指南，一本覆盖Linux系统调用的参考手册和一本书写灵活、高效的代码的权威指南。它有趣而且简单易读，不管你每天的在哪个系统级别编写代码，这本书将教你书写更好代码的诀窍。

阅读对象和假设 在本书中，我们假定读者具有以下的预备知识，熟悉C语言编程和Linux编程环境但不需要精通每个主题。如果你还没有读过任何与C语言相关的书籍，例如Brian . Kerighan 和 Dennis M.Ritchie的经典杰作《C程序设计语言》(Prentice Hall出版，同时也以K&R而为人们所熟悉)。那么我还是高度的建议您先找一本看看吧。如果你不能容忍Unix的文本编辑器，那么选择Emacs和vim这两款通用和广为认可的编辑器吧--任意一个都满足你的工作。你还需要熟悉gcc,gdb,make 等等的基本知识。大量的关于工具和Linux编程实践的书已经早已面世，本书的的后头还列出了一些有用的参考文献。

对于读者关于Unix或者Linux系统编程方面的预备知识，我已经对做最低限度的假设了。这本书将从相关的背景开始，伴随着基础知识，然后慢慢展开到大多数的高级接口和优化技巧上。我希望所有级别的读者都将找到这本书有价值的地方，学到新的东西。在写这本书的过程中，我一直这么坚持这个理念。

对于说服力和读者的动机，我没有做任何假定。底层系统工程师期望能够编写更好的代码这显然是我们的目标，同样那些渴望有一个可靠而健壮的根基的更高系统层次的程序员也将发现这里有很多的乐趣。仅仅因为好奇而来的黑客们也同样欢迎，这本书应该能满足他们的好奇心。 无论读者要什么和需要什么，这本书将展示一个足够广的知识网络来满足他们--最少有Linux系统编程相关的内容送给大家。

不管你的期望是什么，还是除了以上的内容之外另有想法，我衷心期望您能享受这本书的乐趣。

本书内容

这本书被分成10章，1个附录和1个书目。 第一章，简介和本质概念

本章提供一个简介，呈现给大家一个Linux，系统编程，内核，C库和C编译器的概览，就算是高级用户也最好看看这一章--相信我，没错的。

第二章： 文件I/O

本章介绍Unix环境中最重要的概念--文件和Linux编程模型的基础文件I/O。它涵盖了文件读写和其他文件I/O的基本操作，从一个关于Linux内核实现和文件管理的讨论展开的。

第三章：缓存I/O

本章讨论一个基本文件I/O接口--缓冲区大小管理（buffer size management)的论题和以解决方案的方式概述缓冲区I/O以及详细的介绍标准I/O。

第四章：高级文件I/O

本章和盘托出I/O的三驾马车：高级I/O接口，内存映射和优化技术，以关于避免寻找和其在Linux内核的I/O调度程序中的重要性的讨论为结尾。

第五章：进程管理

本章介绍了Unix第二个最重要的概念--进程和基本进程管理的系统调用系列，包括古老庄严的fork函数。

第六章：高级进程管理

本章继续关于高级进程管理的盛宴，包括实时进程。

第七章：文件和目录管理

本章讨论创建、移动、拷贝、删除和其他的文件和目录管理方法。

第八章：内存管理

本章主要涵盖了内存管理。内容从介绍Unix内存概念开始，如进程地址空间和页，然后继续讨论内存获取和返回内存给内核的接口，最后以内存相关的高级接口结束该章。

第九章：信号量

本章介绍的主要内容是信号。本章开始讨论信号和信号在Unix系统中的重要性，然后介绍信号接口，从基础接口到高级接口。

第十章：时间

本章讨论时间，休眠和时间管理。这里涵盖了从基本的接口到POSIX时钟以及高分辨率的计时器。

附录，相对C语言的GCC扩展

附录回顾了很多gcc和GNU C提供的优化相关的知识，如标记函数为常量，纯函数和内嵌函数的属性。

最后本书以一个书目结束，我非常推荐大家读这个书目，它列出了很多有用的补充资料和一些这里没有涉及到的必备专题的书籍。

本书涵盖到的版本

Linux系统接口可定义为由Linux内核（操作系统的核心），GNU C库（glibc）以及GNU C编译器（gcc-现在正式的称呼为GNU编译器集，但我们只关心C）这个三联体提供的应用二元接口（application binary interface 应用程序和内核或者库之间的接口，见wikipedia，译者注）和应用编程接口（application programming interface，API）。这本书讨论的系统接口是由Linux内核2.6.22，glibc2.5和gcc 4.2共同定义的。这本书中的接口是向后（除了新添加的接口）和向前兼容的。

如果说改进中的操作系统的是一个不断前进的物体，那么Linux就如同一直狂暴的印度豹。 它的进展不是用年月来衡量，而是以天单位。内核和其他模块的高频度发布持续着修改着这个游乐场的形体。没有任何一本书能够跟得上这只永远追求时髦且运动着的怪兽。

然而，系统编程定义的系统编程环境是以里程碑的方式设定的。内核开发人员需要和这些心烦的内容打招呼而不是破坏系统调用，glibc的向前向后兼容性和能够产生跨版本兼容的Linux工具链（特别是C语言工具链）。因此，尽管Linux将持续的前进，Linux系统编程还是保持其稳定，而一本基于系统剪影的书，特别是在Linux开发这个关键点上，依然富有魅力的。最后，我想说的仅仅是，不要担心系统接口的改变，购买这本书吧。

本书使用习惯

接下来介绍的是本书的排版习惯：

斜体

用于强调、新术语、URL链接、外来短语、Unix命令和工具、文件名称、目录名称和路径名称。

固定宽度

表示头文件、变量、属性、函数、类型、参数、对象、宏和其他编程结构。

固定宽度斜体

表示将被用户提供的值替代的文本（如：一个路径名称模块）这个图标表示技巧、建议或者通用的提醒。

本书中大部分代码是放在表格中的简短的代码片段，但是是可用的。他们看上去如下： while (1) { int ret; ret = fork ( ); if (ret == -1) perror ("fork"); }

提供简练而可用的代码片段是一件很为难的事情。没有要求指定的头文件，到处充斥着令人心碎的宏和非法链接。但本书还是选择提供位数可观的简单例子来替代构建少数令人窒息的巨大的程序。

通过这些充分描述且确实好用但又精练清晰的例子，我希望提供一个一读起来就觉得有用的编程指南，在以后还可以作为很好参考。

本书中几乎所有的例子都是独立的。这样你就可以轻易的将它们拷贝到你的编辑器中，把它们安放在确实有用的地方。除非特殊说明，否则所有的代码都不需要指定任何编译参数。（在某些特定的例子，你需要连接指定的库）。我推荐大家按照下面的命令来编译源文件： 

    $gcc -Wall -Wextra -O2 -g -o snippet snippet.c

这将源文件snippets.c编译为可执行的二进制文件snippet，开启了警告检测功能，重要且健全的优化功能和调试功能。本书中的代码必须在没有任何错误和警告的情况下通过这个命令的编译--当然，有可能你需要构建为这个代码片段构建一个编程的骨架先。

当在一节中介绍新的函数时，通常以加强字体的Unix手册格式的方式，看上去如下： 

   #include <fcntl.h>
   int posix_fadvise (int fd, off_t pos, off_t len, int advice);

最上部分是头文件和任何必须的定义，紧随着是函数调用的完整原型。
