2.step3: 将应用程序部署到Tomcat

1)路径问题

典型症状:XXX cannot be found.

注意:build.xml文件中

<fileset dir="${appserver.home}/common/lib">这里的appserver.home是tomcat所在路径.注意确认dir为lib所在的实际路径.

<path location="${appserver.home}/server/lib/catalina-ant.jar"/>这里也是一样,改成catalina-ant.jar所在的实际路径.

build.properties 中

appserver.home=${user.home}/jakarta-tomcat-5.0.28 也要改成实际路径.而且路径符号要写成'/'

2)版本号

<javac destdir="${build.dir}" target="1.3" debug="true"中1.3指的是jdk版本号,注意修改

3)tomcat管理员用户名和密码 

    症状：输入ant install时，出现如下消息：
    BUILD FAILED/home/you/gs/build.xml:44:java.io.IOException: Server returned HTTP 
    response    code: 401for URL: http://localhost:8080/manager/install?path= ...

解决方案：确定build.properties文件中的用户名和密码与具有tomcat-users.xml文件里manager角色的用户名和密码相匹配。

在tomcat-users.xml中添入以下内容即可: 

    <tomcat-users>
      <role rolename="manager"/>
      <role rolename="admin"/>
      <user username="admin" password="tomcat" roles="admin,manager"/>
    </tomcat-users>

这里面我确定匹配了近2个小时,一直没有成功,不知道问题所在,后来重启了一下连接就成功了,立刻无语凝噎.

症状1:编译错误.

控制类名与文件要一致,对大小写敏感.

症状2: 

   "Build Failed: Application Already Exists at Path" 错误
    症状：在终端提示窗口输入ant install时，收到如下消息：
    [install] FAIL - Application already exists at path /GSApp BUILD FAILED
    <JWSDP_HOME>/docs/tutorial/examples/gs/build.xml:82: FAIL - Application already exists
    at   path /GSApp 该应用程序已经安装。如果由于应用程序的安装而对它作出了改动，利用ant reload更新Tomcat中的应用程序。

症状3: 

   症状：在终端提示窗口中输入ant install时，收到下列消息：
   <JWSDP_HOME>/docs/tutorial/examples/gs/build.xml:82: java.net.ConnectException: Connection
 refused

解决方案：Tomcat没有完全启动。等待几分钟，然后尝试再次安装应用程序。
