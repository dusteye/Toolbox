/dev/tty /dev/tty0 /dev/tty* /dev/console的区别

Wikipedia，自由的百科全书

/dev/tty是当前进程的控制终端，对某个进程而言。

/dev/tty0 是当前虚拟控制台

/dev/console 有多种情况。在2.1.71后，/dev/console的主设备号为5，次设备号为1，它在内核初始化过程中，会通过一个register_console的函数登记为用作控制台的终端设备，把一个console数据结构挂到内核中的 console_drivers队列中。此后，内核便会将需要显示的信息输出到所登记的设备上。

关于/dev/tty、/dev/tty0的区别，可以通过这么两个小bash语句看到。


先使用root登入tty1和tty2 在第一个虚拟终端(tty1)下，写上： while true ; do echo "hello" > /dev/tty ; done 可以看到tty1上不停的输出hello，切换到tty2上，看不到hello

然后，在第一个虚拟终端下，运行： while true ; do echo "hello" > /dev/tty0 ; done 可以看到tty1上不停的输出hello，切换到tty2上，也可以看到hello

结论：当前虚拟控制台就是你正在使用的那个虚拟控制台。而当前进程的控制终端就是运行该进程的控制终端。

（似乎在这里控制台和控制终端没有什么区别。）
