在/sbin目录拷贝以下程序：agett、fsck、getty、init、insmod、lsmod、modprobe、mount、shutdown、sulogin、swapoff、swapon、sysctl和udev等。

/bin和/sbin目录下的程序依赖于你想建立系统是什么样的，需要那些功能。更重要的是init程序执行系统初始化时所使用的命令。因此/sbin目录下的mount、agett、init和insmod等是不可缺少的。

现在根据/bin和/sbin目录下的程序，拷贝所需要的库文件。比如： 

 [arlequin@localhost ~]$ ldd /bin/bash 
         libtermcap.so.2 => /lib/libtermcap.so.2 (0x00a3b000)
         libdl.so.2 => /lib/libdl.so.2 (0x0095c000)
         libc.so.6 => /lib/tls/libc.so.6 (0x0080b000)
         /lib/ld-linux.so.2 (0x007f2000)

可以看到bash需要/lib目录下的libtermcap.so.2、libdl.so.2、/tls/libc.so.6和ld-linux.so.2库文件。这里要注意这些都是链接，它们指向了实际的库文件。比如： 

 [arlequin@localhost ~]$ ll /lib/libdl*
 -rwxr-xr-x  1 root root 15324 Aug 19  2005 /lib/libdl-2.3.4.so
 lrwxrwxrwx  1 root root    14 Mar 25 17:37 /lib/libdl.so.2 -> libdl-2.3.4.so

libdl-2.3.4.so才是实际执行需要的库文件。所以在拷贝时对cp命令加上-dp参数，保留文件的属性。同时拷贝链接的实际库文件。

建立/etc

/etc目录的建立是最复杂的。它涉及到了init初始化系统的inittab，login使用的group、passwd和shadow等文件。

首先是inittab文件，最简单的形式因该是这样的： 

 # Level to run in
 id:2:initdefault:

 # Boot-time system configuration/initialization script.
 si::sysinit:/etc/init.d/rcS

 # Runlevel 2,3: getty on virtual consoles
 # Runlevel   3: getty on terminal (ttyS0) and modem (ttyS1)
 1:23:respawn:/sbin/getty tty1 VC linux
 2:23:respawn:/sbin/getty tty2 VC linux
 3:23:respawn:/sbin/getty tty3 VC linux
 4:23:respawn:/sbin/getty tty4 VC linux

上面的文件包含三样内容：系统的运行级别（id:2:initdefault:）、启动时的初始化脚本（si::sysinit:/etc/init.d/rcS）和产生登陆终端（1:23:respawn:/sbin/getty tty1 VC linux）。本文件的具体信息可以参看从加电到SHELL或man文档。

rcS初始化脚本的基本工作时：

1. 设定预设路径

2. 设定网络状态

3. 激活系统的置换空间（简单的可以想成是虚拟内存）

4. 检查档案系统：这个时候系统会去检查一些可能会存在的目录，例如 /fsckoptions 与 /forcfsck 及 /fastboot 等等的目录，当你的系统有不正常关机的现象时（例如突然的断电等等），那么 Linux 将会自动的强制去检查 root 的档案系统 （ checking root filesystems ）。5. 周边设定与系统设定的参数 ( /proc )

6. 设定 Plug and Play ( PNP )的一些参数资料

7. 清除 /etc/mtab

8. mount root 及 /proc 档案系统

9. 决定是否使用模块, 加载模块

10. 检查档案系统 ：与前面相同的，当前面的 /fastboot 检查的旗标开启之后，就会以 fsck 检查你的其它扇区啰！ ( 使用 fsck )

11. 挂上其它档案系统

12. 设定 console 字型

13. 打开 quota

14. 清除不必要的档案, 如 lock, pid

15. 设定 clock

16. serial port 初始化

17. 将开机讯息经由 dmesg 放入 /var/log/dmesg 中

由于还不清楚那些是必须的，凭直观感觉，第1步和第8步是必须的。于是建立以下rcS文件（此文件是从Slax Linux中拷贝过来，并修改后的）： 

 PATH=/sbin:/usr/sbin:/bin:/usr/bin

 /sbin/mount -v proc /proc -n -t proc

 # Initialize udev to manage /dev entries for 2.6.x kernels:
 if [ -x /etc/rc.d/rc.udev ]; then
    /etc/rc.d/rc.udev
 fi

 if [ $READWRITE = "no" ]; then
   # Remount the root filesystem in read-write mode
   echo "Remounting root device with read-write enabled."
   /sbin/mount -w -v -n -o remount /
 fi

 #echo "tmpfs / tmpfs rw 0 0">/etc/mtab
 /sbin/mount -f proc /proc -t proc
 /sbin/mount -f sysfs /sys -t sysfs
 
 /sbin/mount -a -v -t nonfs,nosmbfs,noproc

那就需要拷贝/etc/pam.d等相关的配置文件。由于涉及到很多文件，凡是关于pam，security的文件我都拷贝了，但是依然在登陆时显示（none）login：然后无论用什么用户也无法登陆。于是考虑删除passwd、shadow等文件，问题依然。最后干脆将login删除，但是这次直接无法登陆。

最后在LFS手册上看到这可能和/lib/libnss_files*和/etc/nsswitch.conf有关，没有具体试验，只是猜想。

问题的解决，TinyLogin。它的使用非常简单。只是在编译之前要修改配置文件，USE_SYSTEM_PWD_GRP和USE_SYSTEM_SHADOW要设成false。同时还可以编译静态的。编译完成后，安装套件，因为安装程序必须TinyLogin设成setuid的权限，必须以root的身份来安装。执行make install后会建立_intall文件夹，用里面对应的命令替换原来的就可以了。

需要解决的问题

1.Linux启动时的初始化

Linux启动时会用init来初始化系统，首先是init读取/etc/inittab文件，根据内容来初始化系统，其中在inittab中有这么一句： 

 # System initialization.
 si::sysinit:/etc/rc.d/rc.sysinit

表示按:/etc/rc.d/rc.sysinit的内容来初始化。rc.sysinit是一个bash脚本，它主要完成的工作有：激活交换分区，检查磁盘，加载硬件模块以及其它一些需要优先执行任务。RedHat的此文件有1000多行，Ubuntu的是一系列的脚本。我试图改过一些，但还没成功。 问题是：到底要初始化那些东西？那些是必要的？与顺序有关吗？
