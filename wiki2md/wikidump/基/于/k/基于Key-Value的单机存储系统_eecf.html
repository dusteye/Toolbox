项目要求

  • 项目： 单机key-value存储系统
  • 应用场景： 互联网企业中经常要存储各种较大的相对静态的数据，例如图片、视频、音乐等类型的文件，分布式key-value系统就是用来解决这类存储需求的，单机key-value系统是整个系统的基础。
  • 思考问题：
    1. 直接用数据库MySQL存储会有什么问题 ？
    2. 直接在文件系统上存储各个文件会有什么问题 ？
    3. 给出为什么需要单机key-value存储的原因
  • 数据规模:单机存储 1000w条记录， 平均大小100K, 单机存储1T数据, 单条最大长度5Mb
  • 响应时间:平均单次请求在50ms内完成 (SATA硬盘)
  • 接口: put(key, value) , get(key), delete(key)
  • 代码行估计: 5000行
  • 时间要求：1个月内完成
  • 备注: 10月份回学校的时候和大家交流沟通进展和相关技术情况，希望对硬盘/SSD的存储特性有较多了解，并能根据这些特性在单机系统设计的时候有所考虑。

项目管理

  • C code style:
    • http://www.kroah.com/linux/talks/ols_2002_kernel_codingstyle_talk/html/mgp00005.html
    • http://www.kernel.org/doc/Documentation/CodingStyle
  • svn checkout https://smcached.googlecode.com/svn/trunk/ kvs --username 用户名。 在源代码中可以找到：
    • 总体设计文档
    • Library&ClientServer源代码 相关单元测试的代码
    • 如何使用KVS 相关的代码和文档。
  • SVN使用，cliff总结了2条：
    1. “每天开始开发工作前，请及时update代码”
    2. “未调试通过的代码，请在测试项目中进行调试，不要提交到svn”,通过的代码目前指同时满足：
      • Coding Style
      • 编写或完善相关的　测试代码
      • 通过单元测试的代码，cgov
      • 和相关文档的描述一致
