Coding PHP With HTML

PHP 和 HTML 混合在一起使页面显的十分凌乱,布局也不十分明了,调试起来也就十分的不方便,而且转义,输出等 增加了敲击键盘的次数,增加了辐射,十分有必要解决这个问题. 下面是我们在编程中总结的小技巧,为了您和项目的健康,有必要重视一下.

代码重用

  • include AND require

可以用他们来包含大片的HTML代码和函数库，或相对比较静态的内容。由几点要注意

PHP手册：“当一个文件被包含（include or require）时，语法解析器在目标文件的开头脱离 PHP 模式并进入 HTML 模式，到文件结尾处恢复。由于此原因，目标文件中应被当作 PHP 代码执行的任何代码都必须被包括在有效的 PHP 起始和结束标记之中。”

因此，php将把?>后面的空白字符也输出（输出到浏览器），如果在以后引用header()或XAJAX的一些代码，将会出现错误，因为他们不允许在他们之前由其他输出，可用ob_*函数将这些空白输出收取。

  • 函数

我认为在函数中尽量避免php和html的混合，将html的输出分两部分，一是获取变量值，二将变量值输出到html。 类似 <?php echo $var=VarValue?> 这样也许会增加代码的数量，个人认为这样容易修改UI,而且调试比较方便。
