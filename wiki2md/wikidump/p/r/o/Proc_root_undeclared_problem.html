Proc root undeclared problem

Wikipedia，自由的百科全书

在http://www.pdf-search-engine.com/The%20Linux%20Kernel%20Module%20Programming%20Guide-html-tldp.org/LDP/lkmpg/2.6/lkmpg.html一文中介绍了Linux模块开发的过程。其中有一部分是关于Linux下如何写模块挂载/proc目录下面。

在该文章的例子中，http://www.pdf-search-engine.com/The%20Linux%20Kernel%20Module%20Programming%20Guide-html-tldp.org/LDP/lkmpg/2.6/lkmpg.html#31 该页的例子里面，如果直接进行编译，会出现如下问题： 

...hello.c:45: error: ‘proc_root’ undeclared (first use in this function)
...hello.c:45: error: (Each undeclared identifier is reported only once
...hello.c:45: error: for each function it appears in.)

而google到的解决方案如下为 

 http://www.digitalruin.net/node/43

实际上就是将proc_root的位置替换为NULL

而替换后为什么没有问题：

跟踪以下函数即可得到答案： http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=495335 这里会有简单的介绍。

而实际上是 

  create_proc_entry-> __create_
  remove_proc_entry(file: fs/proc/generic.c)->xlate_proc_name
  该函数会判断挂载的父母录是否为空，如果为空，则使用默认的/proc。

因此，直接将原来的 proc_root 修改为 NULL 是没有问题的。
