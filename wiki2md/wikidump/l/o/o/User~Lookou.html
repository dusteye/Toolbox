Framebuffer --Personal Material

Framebuffer相当于一种驱动程序接口,这种接口将显示设备抽象为帧缓冲区。用户可以将它看成是显示内存的一个映像，将其映射到进程地址空间之后，就可以直接进行读写操作，而写操作可以立即反应在屏幕上。该驱动程序的设备文件一般是 /dev/fb0、/dev/fb1 等等。 在本人机器1024X768-24位情况下,清屏方法如下: 

   dd if=/dev/zero of=/dev/fb0 bs=4096 count=768

以上的清屏由块的大小和数目共同决定bs为块,每块为八位,而count为块的数目,两者之积必需和你的屏幕的总位数相等.

通过 FrameBuffer 设备，还可以获得当前内核所支持的加速显示卡的类型（通过固定信息得到），这种类型通常是和特定显示芯片相关的。比如目前最新的内核（2.6.17）中，就包含有对 S3、Matrox、nVidia、3Dfx 等等流行显示芯片的加速支持。在获得了加速芯片类型之后，应用程序就可以将 PCI 设备的内存I/O（memio）映射到进程的地址空间。这些 memio 一般是用来控制显示卡的寄存器，通过对这些寄存器的操作，应用程序就可以控制特定显卡的加速功能.

PCI 设备可以将自己的控制寄存器映射到物理内存空间，而后，对这些控制寄存器的访问，给变成了对物理内存的访问。因此，这些寄存器又被称为memio。一旦被映射到物理内存，Linux 的普通进程就可以通过 mmap 将这些内存 I/O 映射到进程地址空间，这样就可以直接访问这些寄存器了。由于各个PCI设备不同,因此需要针对不同的芯片类型编写不同的用来完成填充矩形的函数.

而对于采用文件操作的方式,将屏幕空间清成某一颜色. 程序如下: 

#include<stdio.h>
#include<sys/types.h>
#include<sys/stat.h>
#include<fcntl.h>
#include <sys/mman.h>
int main()
{
    int fb;
    unsigned char* fb_mem; 
    fb = open("/dev/fb0",O_RDWR);
    fb_mem = mmap(NULL, 1024*768, PROT_READ|PROT_WRITE,MAP_SHARED,fb,0);
    memset(fb_mem, 0xFF, 1024*768);
    munmap(fb_mem,1024*768);
    return 1;
}
